# 安全政策

## 报告漏洞

**请不要通过公开的 GitHub issue 报告安全漏洞。**

相反，请通过电子邮件报告至：**security@gsd.build**（如果邮件弹跳，可在 Discord/Twitter 上私信 @glittercowboy）

包括：
- 漏洞描述
- 复现步骤
- 潜在影响
- 任何建议的修复（可选）

## 响应时间线

- **确认**：48小时内
- **初步评估**：1周内
- **修复时间线**：取决于严重程度，但我们力求：
  - 严重：24-48小时
  - 高：1周
  - 中/低：下一个版本

## 范围

GSD 代码库中可能导致的安全问题：
- 在用户机器上执行任意代码
- 暴露敏感数据（API 密钥、凭据）
- 破坏生成计划/代码的完整性

## 认可

我们赞赏负责任的披露，并在发行说明中提及报告者（除非你希望保持匿名）。
