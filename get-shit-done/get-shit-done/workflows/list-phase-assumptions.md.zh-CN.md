<purpose>
在规划之前展示 Claude 对阶段的假设使用户能够尽早纠正误解。

与 discuss-phase 的关键区别：这是对 Claude 思考的分析，不是对用户已知内容的收集。无文件输出 — 纯粹对话式以促进讨论。
</purpose>

<process>

<step name="validate_phase" priority="first">
阶段号：$ARGUMENTS（必需）

**如果参数缺失：**

```
错误：需要阶段号。

用法：/gsd:list-phase-assumptions [阶段号]
示例：/gsd:list-phase-assumptions 3
```

退出工作流。

**如果提供了参数：**
验证路线图中的阶段存在：

```bash
cat .planning/ROADMAP.md | grep -i "Phase ${PHASE}"
```

**如果找不到阶段：**

```
错误：在路线图中找不到阶段 ${PHASE}。

可用阶段：
[从路线图列出阶段]
```

退出工作流。

**如果找到阶段：**
从路线图解析阶段详情：

- 阶段号
- 阶段名称
- 阶段描述/目标
- 任何提到的范围详情

继续到 analyze_phase。
</step>

<step name="analyze_phase">
基于路线图描述和项目上下文，在五个领域识别假设：

**1. 技术方法：**
Claude 会使用什么库、框架、模式或工具？
- "我会使用 X 库，因为..."
- "我会遵循 Y 模式，因为..."
- "我会将其结构化为 Z，因为..."

**2. 实现顺序：**
Claude 会先构建什么、第二、第三？
- "我会从 X 开始，因为它是基础"
- "然后 Y，因为它依赖 X"
- "最后 Z，因为..."

**3. 范围边界：**
Claude 的解释中什么包含什么排除？
- "此阶段包括：A、B、C"
- "此阶段不包括：D、E、F"
- "边界模糊：G 可能任一方向"

**4. 风险领域：**
Claude 期望复杂性或挑战在哪里？
- " tricky 部分是 X，因为..."
- "潜在问题：Y、Z"
- "我要注意..."

**5. 依赖：**
Claude 假设什么存在或需要到位？
- "这假设了之前阶段的 X"
- "外部依赖：Y、Z"
- "这将被...使用"

对不确定性要诚实。用置信度标记假设：
- "相当有信心：..."（从路线图中明确）
- "假设：..."（合理推断）
- "不清楚：..."（可能多种方式）
</step>

<step name="present_assumptions">
以清晰、可扫描的格式呈现假设：

```
## 我对阶段 ${PHASE} 的假设：${PHASE_NAME}

### 技术方法
[关于如何实现的假设列表]

### 实现顺序
[关于排序的假设列表]

### 范围边界
**包含：** [包含的内容]
**不包含：** [排除的内容]
**模糊：** [可能任一方向的内容]

### 风险领域
[预期的挑战列表]

### 依赖
**来自先前阶段：** [需要的]
**外部：** [第三方需求]
**供给：** [未来阶段从此需要什么]

---

**您怎么看？**

这些假设准确吗？让我知道：
- 我哪些对了
- 我哪些错了
- 我遗漏了什么
```

等待用户响应。
</step>

<step name="gather_feedback">
**如果用户提供纠正：**

确认纠正：

```
关键纠正：
- [纠正 1]
- [纠正 2]

这显著改变了我的理解。[总结新理解]
```

**如果用户确认假设：**

```
假设已验证。
```

继续到 offer_next。
</step>

<step name="offer_next">
呈现下一步：

```
接下来是什么？

1. 讨论上下文 (/gsd:discuss-phase ${PHASE}) — 让我问您问题来建立全面上下文
2. 规划此阶段 (/gsd:plan-phase ${PHASE}) — 创建详细执行计划
3. 重新审视假设 — 我会使用您的纠正重新分析
4. 现在完成
```

等待用户选择。

如果"讨论上下文"：注意 CONTEXT.md 将合并此处讨论的任何纠正
如果"规划此阶段"：了解假设已被理解后继续
如果"重新审视"：使用更新理解返回 analyze_phase
</step>

</process>

<success_criteria>
- 阶段号相对于路线图验证
- 在五个领域展示假设：技术方法、实现顺序、范围、风险、依赖
- 适当时标记置信度级别
- "您怎么看？" 提示已呈现
- 用户反馈已确认
- 提供清晰的下一步
</success_criteria>
