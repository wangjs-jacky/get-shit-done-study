<guidelines>

**需求格式：**
- ID：`[类别]-[数字]`（AUTH-01、CONTENT-02、SOCIAL-03）
- 描述：以用户为中心、可测试、原子化的
- 复选框：仅用于 v1 需求（v2 尚未可执行）

**类别：**
- 从研究 FEATURES.md 类别派生
- 与领域约定保持一致
- 常见：认证、内容、社交、通知、审核、支付、管理

**v1 vs v2：**
- v1：已承诺的范围，将包含在路线图阶段中
- v2：已确认但推迟，不包含在当前路线图中
- v2 → v1 的移动需要更新路线图

**范围外：**
- 明确排除并说明理由
- 防止以后出现"为什么不包含 X？"
- 研究中的反功能应包含在此处并带有警告

**可追溯性：**
- 最初为空，在路线图创建过程中填充
- 每个需求映射到一个阶段
- 未映射的需求 = 路线图缺口

**状态值：**
- 待定：未开始
- 进行中：阶段处于活跃状态
- 完成：需求已验证
- 阻塞：等待外部因素

</guidelines>

<evolution>

**在每个阶段完成后：**
1. 将覆盖的需求标记为完成
2. 更新可追溯性状态
3. 记录任何范围变化的需求

**在路线图更新后：**
1. 验证所有 v1 需求仍然映射
2. 如果范围扩大，添加新需求
3. 如果范围缩小，将需求移至 v2/范围外

**需求完成标准：**
- 当以下条件满足时，需求为"完成"：
  - 功能已实施
  - 功能已验证（测试通过，人工检查完成）
  - 功能已提交

</evolution>

**定义日期：** 2025-01-14
**核心价值：** 用户可以与有共同兴趣的人分享和讨论内容

- [ ] **AUTH-02**: 用户注册后收到邮箱验证
- [ ] **AUTH-03**: 用户可以通过邮箱链接重置密码
- [ ] **AUTH-04**: 用户会话在浏览器刷新时保持

- [ ] **PROF-01**: 用户可以创建带显示名称的个人资料
- [ ] **PROF-02**: 用户可以上传头像图片
- [ ] **PROF-03**: 用户可以撰写简介（最多 500 字符）
- [ ] **PROF-04**: 用户可以查看其他用户的个人资料

### 内容

- [ ] **CONT-01**: 用户可以创建文本帖子
- [ ] **CONT-02**: 用户可以上传带图片的帖子
- [ ] **CONT-03**: 用户可以编辑自己的帖子
- [ ] **CONT-04**: 用户可以删除自己的帖子
- [ ] **CONT-05**: 用户可以查看帖子动态流

### 社交

- [ ] **SOCL-01**: 用户可以关注其他用户
- [ ] **SOCL-02**: 用户可以取消关注用户
- [ ] **SOCL-03**: 用户可以点赞帖子
- [ ] **SOCL-04**: 用户可以评论帖子
- [ ] **SOCL-05**: 用户可以查看活动动态（关注用户的帖子）

## v2 需求

### 通知

- **NOTF-01**: 用户收到应用内通知
