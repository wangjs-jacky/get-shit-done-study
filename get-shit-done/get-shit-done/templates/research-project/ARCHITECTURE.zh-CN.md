<guidelines>

**系统概述：**
- 使用 ASCII 方框图（├── └── │ ─ 仅用于结构可视化）
- 显示主要组件及其关系
- 不要过度详细化 - 这是概念性的，不是实现性的

**项目结构：**
- 具体说明文件夹组织
- 解释分组的理由
- 符合所选技术栈的惯例

**模式：**
- 在有帮助的地方包含代码示例
- 诚实地解释权衡
- 注意在小项目中某些模式可能过于复杂

**扩展性考虑：**
- 保持现实 - 大多数项目不需要扩展到百万级规模
- 关注"什么先损坏"而非理论限制
- 避免过早优化建议

**反模式：**
- 针对特定领域
- 包含正确的替代方案
- 有助于在实施过程中预防常见错误

</guidelines>
