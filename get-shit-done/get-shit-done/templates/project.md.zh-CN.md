# PROJECT.md 模板

用于 `.planning/PROJECT.md` — 活跃的项目上下文文档。

<template>

```markdown
# [项目名称]

## 项目简介

[当前准确的描述 — 2-3 句话。该产品做什么？为谁服务？
使用用户的语言和框架。每当现实与此描述出现偏差时更新。]

## 核心价值

[最重要的那件事。如果其他一切都失败了，这个必须工作。
一句在面临权衡时驱动优先级的话。]

## 需求

### 已验证

<!-- 已发布并确认有价值。 -->

（暂无 — 发布以验证）

### 活跃中

<!-- 当前范围。正在向这些目标构建。 -->

- [ ] [需求 1]
- [ ] [需求 2]
- [ ] [需求 3]

### 范围外

<!-- 明确边界。包含推理以防止重新添加。 -->

- [排除项 1] — [原因]
- [排除项 2] — [原因]

## 上下文

[影响实施的背景信息：
- 技术环境或生态系统
- 相关的先前工作或经验
- 用户研究或反馈主题
- 需要解决已知问题]

## 约束条件

- **[类型]**: [什么] — [为什么]
- **[类型]**: [什么] — [为什么]

常见类型：技术栈、时间线、预算、依赖、兼容性、性能、安全

## 关键决策

<!-- 限制未来工作的决策。在整个项目生命周期中添加。 |

| 决策 | 理由 | 结果 |
|------|------|------|
| [选择] | [为什么] | [✓ 良好 / ⚠️ 重新审视 / — 待定] |

---
*最后更新：[日期] 在 [触发事件] 之后*
```

</template>

<guidelines>

**项目简介：**
- 产品的当前准确描述
- 2-3 句话描述它的功能和目标用户
- 使用用户的词汇和框架
- 当产品发展超出此描述时更新

**核心价值：**
- 最重要的事情
- 其他都可以失败；这个不能失败
- 在面临权衡时驱动优先级
- 很少改变；如果改变，这是一个重大转折

**需求 — 已验证：**
- 已发布并证明有价值的需求
- 格式：`- ✓ [需求] — [版本/阶段]`
- 这些已锁定 — 改变它们需要明确讨论

**需求 — 活跃中：**
- 当前正在构建的范围
- 这些是假设，直到发布并验证
- 发布后移至已验证，如果无效则移至范围外

**需求 — 范围外：**
- 我们不明确构建的边界
- 总是包含理由（防止以后重新添加）
- 包括：已考虑并拒绝、推迟到未来、明确排除

**上下文：**
- 影响实施决策的背景
- 技术环境、先前工作、用户反馈
- 需要解决的已知问题或技术债务
- 随着新上下文的出现而更新

**约束条件：**
- 实施选择的硬性限制
- 技术栈、时间线、预算、兼容性、依赖
- 包含"为什么" — 没有理由的约束会受到质疑

**关键决策：**
- 影响未来工作的重要选择
- 在整个项目生命周期中添加决策
- 当结果已知时跟踪：
  - ✓ 良好 — 证明正确的决策
  - ⚠️ 重新审视 — 可能需要重新考虑的决策
  - — 待定 — 评估为时过早

**最后更新：**
- 总是记录文档何时以及为何更新
- 格式：`在阶段 2 之后` 或 `在 v1.0 里程碑之后`
- 触发内容是否仍然准确的审查

</guidelines>

<evolution>

PROJECT.md 在整个项目生命周期中发展。

**在每个阶段转换后：**
1. 需求失效？→ 移至范围外并说明原因
2. 需求得到验证？→ 移至已验证并引用阶段
3. 出现新需求？→ 添加到活跃中
4. 有需要记录的决策？→ 添加到关键决策
5. "项目简介"仍然准确？→ 如果有偏差则更新

**在每个里程碑后：**
1. 检查所有部分
2. 检查核心价值 — 仍然是正确的优先级？
3. 审查范围外 — 理由仍然有效？
4. 用当前状态更新上下文（用户、反馈、指标）

</evolution>

<brownfield>

对于现有代码库：

1. **首先映射代码库** 通过 `/gsd:map-codebase`

2. **从现有代码推断已验证需求**：
   - 代码库实际做什么？
   - 建立了哪些模式？
   - 什么明显有效且被依赖？

3. **从用户收集活跃需求**：
   - 呈现推断的当前状态
   - 询问他们接下来想要构建什么

4. **初始化：**
   - 已验证 = 从现有代码推断
   - 活跃 = 用户对此工作的目标
   - 范围外 = 用户指定的边界
   - 上下文 = 包括当前代码库状态

</brownfield>

<state_reference>

STATE.md 引用 PROJECT.md：

```markdown
## 项目参考

参见：.planning/PROJECT.md（更新于 [日期]）

**核心价值：** [来自核心价值部分的一句话]
**当前重点：** [当前阶段名称]
```

这确保 Claude 读取当前的 PROJECT.md 上下文。

</state_reference>
