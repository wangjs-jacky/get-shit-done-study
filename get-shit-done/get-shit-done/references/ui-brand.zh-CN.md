## 阶段横幅

用于主要工作流转换。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 GSD ► {阶段名称}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**阶段名称（大写）：**
- `QUESTIONING`
- `RESEARCHING`
- `DEFINING REQUIREMENTS`
- `CREATING ROADMAP`
- `PLANNING PHASE {N}`
- `EXECUTING WAVE {N}`
- `VERIFYING`
- `PHASE {N} COMPLETE ✓`
- `MILESTONE COMPLETE 🎉`

## 检查点框

需要用户操作。62 字符宽度。

```
╔══════════════════════════════════════════════════════════════╗
║  检查点：{类型}                                             ║
╚══════════════════════════════════════════════════════════════╝

{内容}

──────────────────────────────────────────────────────────────
→ {操作提示}
──────────────────────────────────────────────────────────────
```

**类型：**
- `检查点：需要验证` → `→ 输入 "approved" 或描述问题`
- `检查点：需要决策` → `→ 选择：选项-a / 选项-b`
- `检查点：需要操作` → `→ 完成后输入 "done"`

---

## 状态符号

```
✓  完成 / 通过 / 已验证
✗  失败 / 缺失 / 被阻塞
◆  进行中
○  待处理
⚡ 自动批准
⚠  警告
🎉 里程碑完成（仅在横幅中）
```

## 下一步块

总是在主要完成时出现。

```
───────────────────────────────────────────────────────────────

## ▶ 下一步

**{标识符}: {名称}** — {一行描述}

`{可复制的命令}`

<sub>`/clear` 先 → 刷新上下文窗口</sub>

───────────────────────────────────────────────────────────────

**也可选择：**
- `/gsd:备选-1` — 描述
- `/gsd:备选-2` — 描述

───────────────────────────────────────────────────────────────
```

## 反模式

- 不同的框/横幅宽度
- 混合横幅样式（`===`、`---`、`***`）
- 在横幅中省略 `GSD ►` 前缀
- 随机表情符号（`🚀`、`✨`、`💫`）
- 完成后缺少下一步块
