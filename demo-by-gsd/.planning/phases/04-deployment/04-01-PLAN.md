  <action>
    Create the GitHub Actions workflow file at `.github/workflows/deploy.yml` with the following structure:

    ```yaml
    name: Deploy to GitHub Pages

    on:
      # Trigger on push to main branch (DEPL-02)
      push:
        branches: [main]
      # Allow manual trigger from Actions tab
      workflow_dispatch:

    # Required permissions for GitHub Pages deployment (DEPL-03)
    permissions:
      contents: read
      pages: write
      id-token: write

    jobs:
      build:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout your repository using git
            uses: actions/checkout@v6
          - name: Install, build, and upload your site
            uses: withastro/action@v5
            # Uses default Node 22 which matches project requirement (>=22.12.0)

      deploy:
        needs: build
        runs-on: ubuntu-latest
        environment:
          name: github-pages
          url: ${{ steps.deployment.outputs.page_url }}
        steps:
          - name: Deploy to GitHub Pages
            id: deployment
            uses: actions/deploy-pages@v4
    ```

    **Key points:**
    - Use `actions/checkout@v6` (latest stable)
    - Use `withastro/action@v5` - Astro official action (DEPL-03)
    - Use `actions/deploy-pages@v4` - GitHub official Pages deployment (DEPL-01)
    - Permissions block is REQUIRED for GitHub Pages deployment
    - `workflow_dispatch` enables manual triggering for testing

    **Do NOT:**
    - Do not add test steps (optional enhancement, not v1 requirement)
    - Do not use deprecated actions (peaceiris/actions-gh-pages)
    - Do not forget the `permissions` block
