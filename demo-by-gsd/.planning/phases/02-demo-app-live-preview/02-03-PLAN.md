  <action>
Optimize the global.css file to ensure instant style switching through proper CSS variable defaults:

1. **Add Terminal Noir Theme as Default**:
   - Define all CSS variables in :root selector
   - Use the Terminal Noir color scheme from CLAUDE.md
   - This ensures no FOUC on initial load

2. **CSS Variable Structure**:
```css
:root {
  /* Core colors */
  --color-bg: #0a0a0b;
  --color-bg-elevated: #131316;
  --color-text: #e5e5e7;
  --color-text-muted: #6b6b76;
  --color-border: rgba(255, 255, 255, 0.06);

  /* Accent colors - Neon style */
  --color-primary: #00ff88;
  --color-primary-dim: rgba(0, 255, 136, 0.15);
  --color-amber: #ffb800;
  --color-red: #ff4757;
  --color-blue: #00d4ff;

  /* Glass effects */
  --glass-bg: rgba(255, 255, 255, 0.02);
  --glass-border: rgba(255, 255, 255, 0.05);
}

/* Ensure smooth transitions for color changes */
*, *::before, *::after {
  transition: background-color 0.1s ease, color 0.1s ease, border-color 0.1s ease;
}
```

3. **Performance Optimization**:
   - Use CSS custom properties (variables) which update instantly
   - Avoid !important flags that could override injected styles
   - Keep specificity low to allow easy overrides

4. **Import in Layout**:
   - Ensure global.css is imported in Layout.astro or index.astro
   - Verify styles apply on initial page load

This ensures PERF-01 (< 100ms switching) by using CSS variables that browsers update instantly.
